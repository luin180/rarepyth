rarepyth & mace_al_glo

rarepyth is a foundational Python library for computational materials science, providing utilities for VASP job management, MACE machine-learning interatomic potentials, slab/adsorbate structure generation, and FAIRChem integration. mace_al_glo applies rarepyth in a multi-stage active-learning pipeline for MACE-based global optimization of adsorption configurations on catalytic surfaces.

ProjectDescriptionKey Modules
rarepyth Core library: VASP compute engine, MACE optimizers, slab builders, FAIRChem integration, LMDB I/O 15 Python modules across 5 subpackages
mace_al_glo Active learning pipeline: minima hopping → DFT evaluation → selective retraining 8 Python scripts & 6 SLURM templates

Dependencies

mace_al_glo imports from rarepyth via relative path manipulation. Key third-party dependencies shared by both projects:

  • ASE — Atomic Simulation Environment (structure I/O, optimizers, calculators)
  • pymatgen — Python Materials Genomics (Structure, Molecule, symmetry analysis)
  • MACE — Multi-ACE equivariant message-passing neural network
  • FAIRChem / OCP — Open Catalyst Project models and LMDB datasets
  • NumPy, SciPy, Pandas — Numerical & statistical computing

rarepyth.mace.matcher

Physical equivalence classifier for adsorbed structures. Implements a 4-step pipeline distinguishing chemisorption from physisorption, using graph theory and bond-topology analysis.

Workflow

1. Energy Check → 2. Internal Topology → 3. Chemisorption Profile → 4. Z-Height Profile

Steps 3 and 4 are mutually exclusive: two states are equivalent if they match on energy + topology + either chemisorption or physisorption profile.

mace_al_glo — Active Learning Pipeline

Pipeline Overview

MACE Active Learning Global Optimization (AL-GLO) is an iterative 4-stage pipeline that generates adsorption configurations, screens them with MACE minima hopping, evaluates MACE vs DFT predictive quality, and selectively retrains the model on the worst-performing structures.

Step 0
Minima Hopping → Step 1
VASP SP Prep → Step 2
DFT Relaxation → Step 3
MACE Training → Iterate

Iteration tracking: A file named ITER stores the current iteration number. It increments after each training cycle. At iteration 0, the foundation model is mace_mp_medium.model; subsequent iterations use the fine-tuned model from the previous round.

Cross-cluster workflow: The pipeline is designed for heterogeneous HPC environments — MACE runs on GPU nodes (minima hopping, training) while VASP DFT runs on CPU nodes. tar.gz archives shuttle data between clusters.

step0_mh.py — Batch Dispatch of Minima Hopping

Purpose

Generates the Cartesian product of selected slab surfaces and adsorbate molecules, then dispatches each system as a SLURM job for parallel local minima hopping with MACE.

Flow

  1. Validate workbench/slabs/ and workbench/molecules/ exist
  2. Read ITER counter to determine checkpoint path (ft_iter{N-1}.model or mace_mp_medium.model)
  3. Create timestamped work directory: work_YYYY-MM-DD-HH-MM-SS/mh/
  4. 2D strided random sampling: Select slab_batch slabs and molecule_batch molecules
  5. For each slab × molecule pair, create a subdirectory containing:
    • Slab POSCAR, adsorbate molecule file
    • run_mh.py and run_mh.sh templates
    • CHECKPOINT_PATH file pointing to the model
  6. Print SLURM sbatch commands (for manual submission or piping)

Key Parameters

ParameterDefaultDescription
slab_batch1Number of slabs to sample per iteration
molecule_batch3Number of molecules to sample per iteration

step1_sp.py — VASP Single-Point Preparation

Purpose

Extracts the top unique minima from each system's MACE global optimization results and packages them for VASP single-point DFT evaluation on a CPU cluster.

Flow

  1. Read TIMESTAMP_ID to locate current work directory
  2. For each system in work_*/mh/, open FINAL_GLOBAL_MINIMA/FINAL_MINIMA_EE
  3. Select top 20% unique minima (minimum 6) by energy
  4. Copy each selected geometry to work_*/sp/{system}_UNIQUE_{idx}/ with TAG file, vasp_sp.py, vasp_sp.sh, and INCAR.ini
  5. Package all SP directories into vasp_sp_tasks_{timestamp}.tar.gz

Output

  • tar.gz archive for cross-cluster transfer
  • FINAL_MINIMA_EE contains MACE-predicted energies for all unique minima

step2_rlx.py — Relaxation Task Dispatcher

Purpose

Evaluates MACE vs DFT predictive quality using ranking correlation metrics, then selects the worst-performing structures for full VASP DFT relaxation. This is the core active learning decision step.

Evaluation Metrics

MetricDescription
Spearman's ρRank correlation between MACE and DFT energy rankings
Kendall's τRank correlation (concordant/discordant pairs)
Top-1 hit rateDoes MACE rank the DFT global minimum as #1?
Top-3 hit rateIs the DFT global minimum in MACE's top 3?
MACE rank of DFT min1-indexed rank of the DFT global minimum in MACE's ordering
Local DDE MAEMean absolute error of MACE vs DFT relative energies within the dynamic window from MACE's predicted minimum to DFT minimum

Active Learning Selection

Systems are sorted by LOCAL_DDE_MAE_eV descending. Systems with MAE ≥ 0.15 eV are dispatched for VASP relaxation (up to rlx_batch structures). Each relaxation directory includes vasp_relax.py, vasp_relax.sh, INCAR.ini, POSCAR.vasp, and TAG.

Templates

templates/run_mh.py — Per-System Full Workflow

The most complex template. Orchestrates three phases for each slab+molecule system:

Phase 1: Seed Configuration Generation

  1. Rotate the adsorbate molecule (Euler angles, 45° steps)
  2. Deduplicate rotated molecules via permutation-invariant RMSD
  3. Find filtered adsorption sites on the slab surface
  4. Compute MACE single-point energies for all configurations
  5. Select top 4 lowest-energy seeds

Phase 2: Parallel Minima Hopping

  1. For each of the 4 seeds, create Top{rank}_{idx}_{jdx}/ directory
  2. Generate TAG, HOOKEAN, and POSCAR restraints
  3. Dynamically write sub_run_mh.py calling MultiGlobalOptimizer with 2 strategies:
    • aggressive_600 — 600 minima hopping steps
    • extreme_1200 — 1200 minima hopping steps
  4. Launch all 4 as parallel subprocesses (each spawning 2 workers = 8 total GPU workers)

Phase 3: Final Aggregation

Reads all Top* directory results, performs graph-theory secondary clustering using is_equivalent_state from rarepyth.mace.matcher, outputs deduplicated global minima with energy ranking.
